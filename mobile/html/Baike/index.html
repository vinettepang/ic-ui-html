产品经理

职位描述

工作内容

互联网产品经理是互联网公司中一个常见的职能部门，负责互联网产品的计划和推广，以及互联网产品生命周期演化的推进。职能上，狭义的产品经理只负责产品策划工作，而广义上，产品经理则需同时兼顾产品的策划和运营工作。产品经理在互联网公司中处于核心位置，需要非常强的沟通能力、协调能力、市场洞察力和商业敏感度。最著名的互联网产品经理有百度的俞军，腾讯的张小龙等。产品经理的工作内容主要有以下几点：

  • 负责用户需求的调研、分析、评估和原型设计；
  • 推动、协调产品功能的开发、测试和发布；
  • 协助产品上线以后的 BUG 跟踪、收集改进意见、提供改进方案；
  • 定期对行业相关产品进行评估，并提出功能优化、用户体验升级等方面建议。

岗位要求

产品经理岗可谓是现阶段互联网众多岗位中的“当红炸子鸡”，相对低的 IT 技术要求，无专业限制，重思考的逻辑性，重沟通的效率性，重团队协作的顺畅性，为了更好地完成日常工作，产品经理必须熟练掌握以下技能：

  • 熟练使用Axure，Xmind，Office等软件输出产品文档，产品原型图，流程图，思维导图等。
  • 具备较强的逻辑分析能力，通过问卷，聊天等形式收集用户的需求，分析用户需求。
  • 有一定的市场敏感度，调研竞品，优化产品功能，提升产品的用户体验。
  • 热爱互联网行业，有较强的协作能力，沟通能力以及创新能力，做事严谨认真。

职业发展

职业发展晋升图

大型互联网公司发展比较成熟，各个岗位的职业发展体系也比较完善，所以关于产品经理的职业发展我们会主要以大型公司为背景。应届生进入大型公司后，发展情况都大同小异，一般会从产品助理 / 专员（新人）进阶到产品经理（成熟）， 再进阶到高级产品经理（骨干）。一般每个阶段会经历0.5-2 年，具体的晋升速度取决于产品线发展速度、公司发展情况、个人成长速度和导师水平。

产品助理/专员 0-2年 5-8K／月

在导师的指导下，可以完成相对简单的任务，比如设计一个小的功能并推动上线，或完成一份比较清楚的竞品分析。

产品经理 2-5年 8-20K／月

在导师交代清楚任务的前提下，可以完成有一定复杂度的任务，比如负责某个产品的一个版本迭代并推动上线，或者完成对某个行业的深度调研。

高级产品经理 5年以上 20-50K／月

主动和上级领导沟通战略目标，独立完成复杂任务，比如完成一个全新产品的规划和设计，然后带领团队完成项目并上线，并且需要有方法论和工作经验的产出，既可以指导新人，又可以建立起对团队的影响力。

成为高级产品经理后，将面临两个选择

  • 专家路线（P路线）
  • 管理路线（M路线）

一般而言，BAT级别的公司每个大型产品都需要同时配备至少一名产品专家（产品架构师）和一名管理者（经理）。其中，产品专家具体负责产品深层次的分析和规划，把更多精力放在业务本身上；而管理者要花更多精力去做人事管理，大到人力资源分配，小到请假审批，都要处理。两个通道的晋升难度相当，各自的工作也有很大部分的重合，所以具体选择取决于个人倾向、擅长领域和组织需要。一般而言，更喜欢集中精力钻研业务的同学未来可以考虑走专家路线，而更喜欢和人打交道、领导大家一起完成目标的同学则比较适合走管理路线。

热门问题

Q1:产品经理怎么入门，如何自主学习？

1.首先清楚产品经理的要求是什么？

想要成为一个产品经理，总要先了解企业对产品经理的要求是什么，大部分公司要求的产品经理即功能型产品经理：①设计技能：懂得产品模式的功能结构，掌握功能的策划和设计；②产品软件：熟练使用office、思维导图、原型设计软件、各类产品文档；③产品模式：了解B2C、O2O、SNS等产品的功能结构和运作模式。

2.和产品经理的差距在哪里？如何提高？

从最简单的软件入手。产品相关软件：①使用Axure进行原型设计 ②使用Mindmanager、Xmind绘制思维导图 ③Axure，Visio画流程图 ④项目管理软件：Project，禅道； ⑤办公软件：Office，WPS等；以上软件起码要掌握基础的操作，如利用AXURE会做简单的交互动作，利用VISIO会画简单的流程图，泳道图；利用Peoject会做简单的项目排期，利用Excel会做简单的数据分析。

清楚不同产品模式下不同的功能结构，并且可以根据企业的产品模式，设计出产品原型及文档；这里的建议是： ①从设计开始：对不同产品模式的网站：如B2B,B2C,C2C这三种不同类型的网站，尝试着自己设计出一套产品原型出来，从最简单的一套流程开始，如注册流程、下单流程... ②对比：把自己设计的产品，与真正的网站进行对比，找出自己所完成的流程与真正的、成熟的流程相比下的差距在哪里 ③小结：思考以下这些问题：有哪些细节没有想到？为什么会有这样的差距？自己的思维有哪些盲点？设计这些产品的产品经理是怎么考虑的？ ④反复修正：根据对比的结果找到自己的不足后，给第一次自己做的设计进行修改，再重新进行对比——小结——修改，对作品进行反复的修正。 ⑤如果掌握了相关软件和不同产品模式下的功能设计，那么当个产品专员是足够了的。

3.产品经理必备的素质：

①一定的技术功底或者知识：虽然在知乎上大家一直都在讨论产品经理到底需不需要技术功底，但是掌握一定的编程知识，不但可提升自己的逻辑思维能力，在跟程序员的沟通上，也少了很多问题，所以就算不会编程，还是建议掌握一定的技术知识。 ②一定的运营知识：再好的产品也需要运营，不管是为了产品还是为了更好的同运营部门沟通，运营知识是少不了的； ③一定的沟通技巧：产品经理虽然被称为迷你CEO，但他是一个没有权力的CEO，如何利用公司的各种资源为自己服务，完成产品，那么沟通技巧就不可缺少了； ④一定的设计技巧：在很多公司没有交互设计职位的情况下，产品经理和UI负责了整个产品的设计，不具备一定设计知识的无法把产品经理的工作做到游刃有余。如果以上的标准都达标，那么获得一个产品经理的职位，并不是很困难的事情。

总结起来，产品经理最重要的三条能分别是学习能力、逻辑思维、沟通。而其中，学习能力排在第一位。知乎上有大量关于产品方面的提问和答案，在书店中也有大量关于产品经理的书籍，只要你有心去学习、去关注，掌握相关知识和技能并不是一件什么难事。

Q2:作为一名产品老鸟，你对产品新人有哪些忠告？

写文档、画原型等基本功就不详谈了，这些都是必备的基本功，通过坚持练习就可以提升的。而作为一名PM除了掌握这些实操技能之外，还需提升以下这些方面的能力：

  1. 充分理解业务，深度理解自己负责产品的前后台、业务逻辑、历史版本信息，了解各种关联系统。当需求来临时，能根据目标，迅速给出一个最优的解决方案。
  2. 要多去了解用户，看看用户在说什么。了解了用户才能接地气，产品的更新迭代才能真正提升用户体验。
  3. 提高分析数据的能力。有数据，评审需求时不仅有说服力，还能根据数据判断产品优化的方向，去做一些创新。
  4. 提高项目管理能力。产品经理往往也是项目经理。如果一个产品，在产品阶段有90分，经过设计、前端制作、研发、测试阶段，到上线可能只剩下60分。管理好项目，就是要保证项目按时高质量上线，上线时也能有80分以上。
  5. 学会沟通。需求从诞生到上线，要那么多人经手，不提前沟通好，让项目成员都理解需求，最后测试肯定会乱成一窝。还有，运营、客服、销售等业务人员会经常找你解决各种问题。除了要对自己产品了如指掌，还要时常为团队的成员提供解决的方案。
  6. 坚持学习。除了努力工作，把事情做好，把产品搞好，还要不断学习。有的人工作了3年、5年，但一直在复用经验，工作经验也就相当于1年；有的人工作1年，但处事的方式方法果断老练，不亚于有3、5年工作经验的“老人”，是因为知识体系一直在更新。
  7. 要有创新精神。不是凭空瞎想，也不是看别人做什么就跟风。要根据自己产品特点、业务情况、用户群体，去研究怎么让产品更好地为用户服务，更好地为公司创造价值。

Q3:产品经理需要懂技术吗？懂到什么程度？

产品经理需要懂技术，创业者尤其需要。做产品的，经常需要和开发工程师打交道，而和他们交流通常有两大忌：

1.忌不懂技术

更准确的说，是不能缺乏设计、开发一个互联网产品基本的技术常识，比如至少要清楚一个网站从不存在到能被用户访问，需要哪些必须的环节；也要明白一个App从你的脑海走到用户的手机里，需要经历怎样的过程。。有常识，不一定就能做出好产品，但没常识，就很像在村里呆了半辈子的人乍到城市，一举一动即使小心翼翼，也难以掩盖掉那股突兀和违和感。

2.忌懂技术

我遇到不少工程师喜欢说：“只要产品需求明确，技术上一切都能实现。”这句话听起来相当豪迈，也让产品经理大为放心，觉得技术真是产品的坚强后盾。但其实传递了一个特别糟糕的信号。当工程师这么说的时候，潜台词是：“你弄好你自己的事儿就行了，别来管我！”而且这种说法隐含着一个乐观但显然并不现实的假设：技术是无所不能的，他（掌握技术的人）像神一样，可以实现你的任何愿望，只要你能明确的描述它。至于技术需要懂到什么程度，敝人觉得要是花几个月学的东西就够用一辈子，这买卖也太划算了，尤其是在技术领域。技术的东西是会更新变化的，需要不断地去学习、吸收新的知识，同时知识的汲取也分缓急轻重。马上要用到的、能显著提高现阶段工作效率或者公认是最佳实践方法的知识就先学，否则就可以先把学习计划稍稍延后，尽量不折腾、严格控制投入的时间和精力。

产品经理懂技术，在没资源的时候可以用最低成本把事儿办了，有资源的时候可以把资源用的更有效率。

Q4:高级产品经理和普通产品经理有哪些区别？

1、产品切入点选择的眼光

高级产品经理不仅了解用户和市场，更懂得自身/团队的特长。从而能合理的抓准产品切入点，进入市场，制定战略战术，打败竞争对手。这也是为为什么很多公司，特别是创业公司，产品都是公司老大亲自负责的原因。而普通产品经理更多负责的是执行，协助高级产品经理，保证战略战术在功能细节上的有效开展。

2、把控整体节奏的能力

高级产品经理需要足够的大局观，明白产品要想成功，产品、运营、技术三个核心环节缺一不可。也就是说，在高级产品经理眼里，产品是个整体，什么时候做迭代，什么时候做运营推广，都是有计划、有节奏、按步骤来进行的。如果只会被动地去满足需求和设计，认为自己与产品新功能上线无关的产品经理，只会止步于前，被动地做一个执行者。

3、懂得吸取有价值的经验与教训

产品成功的经验很重要，失败的教训更重要。只有经历了足够多的失败，才能清楚的知道哪里有坑、什么不该做、什么是无用功。不计较产品上一时的得失，关注最后的结果。

4、高人一筹的情商

产品是团队共同努力的结果，所以作为团队核心的产品经理，没有高人一筹的情商，不明白“闲下饭桌上的沟通远比正式邮件上的指责更有效”的道理，一定称不上高级产品经理。
